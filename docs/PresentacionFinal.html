Gestor de Prestamo de Equipos Audiovisuales

Programacion I

Presentacion base lista para exponer o pasar a PowerPoint / Canva.

Problema

Que necesidad resuelve el sistema

  • El control manual de camaras, proyectores y accesorios genera retrasos y desorden.
  • No siempre se sabe que equipo esta disponible o quien lo tiene prestado.
  • Se pierden evidencias sobre devoluciones tardias, observaciones y danos.
Idea

Solucion propuesta

Desarrollar un sistema de gestion que registre usuarios, equipos, prestamos, devoluciones y danos, con una estructura profesional basada en POO, capas, base de datos y arquitectura distribuida.

Camaras Proyectores Accesorios Prestamos Danos
Objetivo

Meta general del proyecto

Crear una solucion en C# para administrar el prestamo de equipos audiovisuales, permitiendo controlar disponibilidad, entregas y devoluciones, con persistencia en MySQL y comunicacion entre cliente y servidor.

POO

Conceptos aplicados

  • Clase abstracta: Equipo
  • Clases derivadas: Camara, Proyector, Accesorio
  • Clases normales: Usuario y Prestamo
  • Constructores: usados en Usuario y Prestamo
  • Sobrecargas: RegistrarDanio y RegistrarDevolucion
Base de Datos

Conexion y persistencia

La aplicacion guarda usuarios, equipos y prestamos en MySQL. El sistema valida disponibilidad, actualiza estados y registra observaciones o danos en cada devolucion.

Tablas principales

Usuarios, Equipos y Prestamos

Beneficio

La informacion no se pierde y puede consultarse en cualquier momento.

Distribuida

Como funciona la arquitectura

  • La API recibe y procesa las solicitudes del sistema.
  • El cliente de consola consume la API para consultar o registrar informacion.
  • La base de datos se gestiona desde la capa de infraestructura.
Funcionalidades

Que puede hacer el sistema

Usuarios

Crear y consultar usuarios.

Equipos

Registrar camaras, proyectores y accesorios.

Prestamos

Crear prestamos solo si el equipo esta disponible.

Devoluciones

Registrar devoluciones y reportar danos.

Valor

Beneficios del proyecto

  • Mejora el control del inventario audiovisual.
  • Reduce errores en prestamos y devoluciones.
  • Aporta trazabilidad sobre danos y mantenimiento.
  • Sirve como base para futuras mejoras como autenticacion o reportes.
